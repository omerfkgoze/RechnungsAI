## Challenges

### Story 4.3 Patch Spike (18 patches — regression from Epic 3's 22→8 trend)

Root cause: Story 4.3 introduced 4 new technical surfaces simultaneously (ZIP writer, CSV builder, Route Handler, archive UI) with **no dedicated spike**. Compare:
- Story 4.1: spike-p3 provided integration sketch → 5 patches
- Story 4.2: prep-p4 provided schema + wiring plan → 6 patches
- Story 4.3: no spike → dev agent built everything from scratch → 18 patches

Key patch categories: CSV formula injection (security), ZIP integer overflow guard (correctness), failed Storage download wrote 0-byte file (audit integrity), migration regex accepted invalid dates, mobile layout `overflow-x` breaking `position: fixed`, `type="date"` missing German masking.

### Date Input UX Regression (GOZE's direct feedback)

`input type="date"` reintroduced in Story 4.3 despite `applyGermanDateMask` pattern being fully shipped in Story 3.5 (`editable-field.tsx:236-248`, `lib/format.ts`). GOZE: *"German users type dates in DD.MM.YYYY format. UI should show this format. Backend/DB can use ISO."* This is now a project rule (see Action A3).

### Mobile Layout Regression — Archive Page

Bottom nav (`position: fixed`) disappeared unless scrolled to page bottom; page overflowed horizontally on mobile. Cause: archive components (filter bar, result list, pagination) missing `min-w-0` / `overflow-x-hidden` / `pb-16` discipline. Reference existed in `dashboard/page.tsx` but was not consulted. Fixed in review patches.

### sessionStartMs → errorCount Always 0 (TD1)

Story 4.2 correctly wired `audit_logs` count query to `<SessionSummary errorCount>`. However, `sessionStartMs = Date.now()` is render-time, so `.gte("created_at", sessionStartMs)` always returns 0 edits for the current "session." The feature works correctly at the data layer but the session tracking model needs rethinking. Deferred.

## Key Insights

1. **Spike = mandatory for new technical surfaces.** New package, Route Handler, or binary format without a prior spike produces 3× the review patches. This is measurable and predictable.

2. **"Pattern first" discipline.** New UI components must reference the closest existing implementation in the story file before writing. Date inputs → `editable-field.tsx:236-248`. Mobile pages → `dashboard/page.tsx` layout.

3. **Smoke test is the last defense before production.** Caught mobile nav regression, date masking regression, and security posture questions that automated tests cannot. Non-negotiable for every story.

4. **DATEV format compliance needs versioned architecture.** GOZE concern: DATEV EXTF format changes over time. Solution (Winston): versioned pure functions in `packages/datev/src/formats/extf-vXXX.ts`. Format update = new file, existing code untouched.

5. **Security patches in compliance code need extra attention.** CSV formula injection and ZIP integer overflow in a GoBD-compliance story are higher severity than in a regular feature story. Spike-first + architecture review helps catch these before implementation.
