## Challenges

### Review Cycles Were Heavy
Story 2.2 had 14 patches; Story 2.3 had 10 patches + 4 post-review UX fixes. Pattern: correct path works, error paths under-specified. Recurring issues:
- DB select error silently swallowed (server returns "not found" instead of real error)
- Revert writes (status flip-back) were fire-and-forget — compensating update errors discarded
- TOCTOU race on `processing` status flip (Story 2.2)
- `alert()` used instead of inline German hint (UX-DR12 violation)

### Smoke Tests Need Better Format
Current format lists steps but lacks: (a) expected output alongside each step, (b) "pass/fail" criteria per check, (c) distinction between developer-relevant psql and user-experience verification. GOZE confirmed: time is not the issue — quality of instructions is.

### Next.js 16 API Drift — Pre-Implementation Surprises
Story 2.2 required 3 pre-implementation fixes before implementation could start (`middleware.ts` → `proxy.ts`, Sentry instrumentation-client, next.config deprecation). These were not story-related but blocked the dev flow.

### Zod v3/v4 Type Depth Explosion
The AI SDK (`ai@6.0.168`) carries a Zod v4 peer dependency while `@rechnungsai/shared` uses Zod v3.25. This causes TypeScript type instantiation depth overflow on `generateObject` + `invoiceSchema`. Current workaround: `as unknown as` cast. Epic 3 schemas building on `invoiceSchema` will inherit this issue unless resolved.

### Sentry Noise
Story 2.3 review found `captureException` was being called for structured `{ success: false }` action results (e.g., 429, validation errors) — these are expected control flow, not exceptions. Fixed in review: `captureException` only for real `catch` throws; `captureMessage` at warning level for structured failures.

## Key Insights and Lessons Learned

1. **Prep sprint model is worth it** — 7 prep tasks done before Epic 2; the epic started with zero blockers. Replicate for Epic 3.
2. **Smoke tests are high-leverage** — GOZE's manual test found bugs unit tests cannot: UX flow problems, React StrictMode double-mount, timing issues. The format needs improvement, not the culture.
3. **Error path reviews need a checklist** — repeated pattern of happy path complete, error paths partial. A standard Server Action review checklist ("every exit path reverts status? catch block not fire-and-forget?") will reduce this category of findings.
4. **`overallConfidence` is now arithmetic mean** — changed from `Math.min` during Story 2.2 (single low-confidence field was dragging the overall badge to 0%). Epic 3 Story 3.4's confidence-based sort uses this mean — not the minimum.
5. **Fire-and-forget applies to uploads too** — Story 2.3 initially blocked `Fertig` while uploads were in-flight (overcorrection). The durability contract is IDB enqueue, not awaited upload. Both upload and extraction are non-blocking.
6. **Module-scoped state (extraction-gate.ts) survives navigation** — intentional; in-flight Server Actions continue server-side after the user navigates away. Queue clears, counters track, but work persists.
