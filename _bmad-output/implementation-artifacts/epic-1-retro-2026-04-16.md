## Challenges

### Story 1.3 — 28 Patches (Largest Review Cycle)
Auth + RLS + Supabase SSR + Server Actions + form primitives all in one story created extreme complexity. Key issues:
- `@supabase/ssr` cookie API shape differed from LLM training data
- `(onboarding)` route group created wrong URL (`/trust` instead of `/onboarding/trust`) — silently broken until Story 1.4
- `zod` v4 vs v3 peer dependency conflict
- Critical: `users_update_self` RLS policy allowed `tenant_id` reassignment (cross-tenant privilege escalation)
- Second review pass found more issues including middleware matcher too broad

### Vitest Deferred Across All 5 Stories
Testing harness deferred every story. Compounding risk: `transpilePackages` pattern means Vitest cannot currently resolve `@rechnungsai/shared` outside Next.js context — needs a build step fix before Vitest can run.

### Browser Smoke Tests Not Executable by Dev Agent
Story 1.2 explicitly noted: *"Dev server / browser smoke test NOT executed by the agent because the dev environment cannot launch an interactive browser session."* This gap persisted through all stories.

### Next.js 16 API Drift
Every story cited this. LLM training data reflected older Next.js APIs; `AGENTS.md` correctly prescribed reading installed docs first — this discipline saved multiple potential mistakes.

---

## Key Insights and Lessons Learned

1. **Large stories create large review cycles** — Story 1.3 scope was too broad. Future stories should be scoped to avoid 5+ distinct technical concerns simultaneously.
2. **Smoke tests saved us even without Vitest** — The manual psql and browser smoke test protocol was effective. Document it more explicitly per story.
3. **Fix-triggered discoveries are valuable** — The middleware fail-closed patch in Story 1.4 revealed a pre-existing RLS bug from Story 1.3. Counter-intuitive: making code stricter surfaced hidden issues.
4. **Spike before complex integrations** — `@supabase/ssr` surprise in Story 1.3 would have been avoidable with a 1-hour spike. Apply same discipline to Vercel AI SDK v5 in Epic 2.
5. **context7 + brave-search MCP for spike tasks** — Use these tools for researching library APIs (especially for rapidly-evolving packages like AI SDKs) before implementation.
