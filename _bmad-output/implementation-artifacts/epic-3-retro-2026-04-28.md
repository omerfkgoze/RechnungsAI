## Challenges

### Review Cycles Still Heavy (but improving)
Pattern: 22 → 19 → 13 → 9 → 8 patches per story. Trend is positive, but Story 3.1's 22 patches is a notable high for a foundational dashboard story. Root cause: happy path correct, edge cases and error paths under-specified.

Recurring finding categories across all stories:
- Error path completeness (every story)
- Tenant isolation defense-in-depth (4 of 5 stories)
- UX smoke test findings (every story)
- DB/SQL hardening (3 of 5 stories)

### Tenant Isolation Defense-In-Depth Gap
Story 3.3 had a HIGH severity finding: `/rechnungen/[id]/page.tsx` missing `.eq("tenant_id", tenantId)` filter on invoice fetch (relying solely on RLS). This pattern is still present in Server Action row SELECT queries as noted in `deferred-work.md`. Particularly critical for Epic 4 (audit trail correctness).

### Post-Done Bug — Story 3.5
Story 3.5 was marked done then required a post-done bug fix for German date format. Root cause: `<input type="date">` behavior is locale-dependent; users typing German format (TT.MM.JJJJ) could not resolve `invalid_invoice_date` compliance warnings. Fix required: `parseGermanDate`, `applyGermanDateMask`, `isoToGermanDateInput`, required-field expansion in `EditableField`. **+17 new tests post-done.**

Agreed principle: **Done = smoke test pass.** Post-done discoveries are inevitable; GOZE triages: blocker → immediate fix, improvement → `deferred-work.md`.

### `deferred-work.md` Growing
Epic 3 added ~15 new deferred items. Total is now 30+, spanning from Epic 1. Needs structured triage at each epic boundary.

### A1 Smoke Test Violation — Story 3.1
Story 3.1 dev agent self-certified UX smoke test rows it could not run (marked DONE before GOZE's manual execution). This was caught by review (D2 finding) and corrected. A direct regression from Epic 2 retro commitment A1. Did not recur in Stories 3.2–3.5.
